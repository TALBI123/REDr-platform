# 2026-05-31 - Agency filtering and pagination report

## Overview
This report documents how pageable pagination and specification based filtering work in the agency module.

## Why Specification
Dynamic filters are hard to model with fixed repository methods. Each optional field would require new methods, which does not scale. Specification solves this by:
- building predicates only for the filters that are present
- composing filters with and/or logic
- keeping query logic in one place for reuse and tests

## Specification interface
`org.springframework.data.jpa.domain.Specification<T>` is a functional interface with:
- `toPredicate(Root<T>, CriteriaQuery<?>, CriteriaBuilder)`

Spring Data calls this method to produce a `Predicate` that is applied to a `CriteriaQuery`.

## Predicate (Criteria API)
`jakarta.persistence.criteria.Predicate` represents a boolean expression in a query. It is used because the Criteria API is strongly typed and composable. Predicates can be combined with `builder.and(...)` or `builder.or(...)` to create complex filters while staying type safe.

## Who implements Predicate
`Predicate` is an interface. The concrete implementations are created by the JPA provider (Hibernate in this project). Spring Data does not implement it. It simply calls `CriteriaBuilder`, which returns provider specific predicate instances. These classes are internal to Hibernate and should not be referenced directly.

## SQL generation flow (detailed)
1. Controller receives query params (example: `name`, `status`, `page`, `size`, `sort`).
2. Spring MVC binds params into `AgencySearchCriteria` and `Pageable`.
3. Service calls `repository.findAll(spec, pageable)`.
4. `SimpleJpaRepository` builds a CriteriaQuery:
   - `CriteriaBuilder cb = entityManager.getCriteriaBuilder()`
   - `CriteriaQuery<Agency> cq = cb.createQuery(Agency.class)`
   - `Root<Agency> root = cq.from(Agency.class)`
   - `Predicate p = spec.toPredicate(root, cq, cb)`
   - `cq.where(p)`
   - apply Sort from `Pageable`
5. JPA creates a `TypedQuery` from the CriteriaQuery.
6. Hibernate translates the criteria tree to SQL with bound parameters.
7. When using `Pageable`, a count query is executed to compute total elements.
8. SQL runs as a prepared statement with `LIMIT` and `OFFSET` for paging.

## Example SQL shape
Data query:
```
select *
from agencies
where lower(name) like ? and status = ?
order by name asc
limit ? offset ?
```
Count query:
```
select count(*)
from agencies
where lower(name) like ? and status = ?
```

## Notes and caveats
- `lower(field) like ...` can bypass indexes; consider a functional index if needed.
- Paging always triggers a count query; heavy filters may be expensive.
- Avoid fetch joins in Specifications used with paging to prevent duplicates.
