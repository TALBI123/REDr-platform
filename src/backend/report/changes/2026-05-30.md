## Workflow details
### Login and JWT workflow
1. `AuthController.login()` receives credentials and calls `AuthService.login()`.
2. `AuthService.login()` validates input, checks lock status, then delegates authentication to Spring Security.
3. `UserDetailsServiceImpl` loads the user from `UserRepository` and returns `CustomUserDetails`.
4. `JwtService` generates the token with `uid`, `role`, and optional `agency_id` claims.
5. `AuthController` returns a response and sets the ACCESS_TOKEN cookie.

### Registration workflow
1. `RegistrationService.register()` creates a `Client` with role `CLIENT` and status `PENDING`.
2. `RegistrationService.registerAgencyManager()` creates the `Agency` then creates an `AgencyManager` with role `AGENCY_MANAGER` and status `PENDING`.
3. `EmailVerificationService` sends a verification email after user persistence.

### Authorization and role workflow
1. `JwtFilter` extracts the JWT, validates expiry, and loads `CustomUserDetails`.
2. `SecurityConfig` applies role checks using `SUPER_ADMIN`, `AGENCY_MANAGER`, and `CLIENT`.
3. `SecurityUtils` exposes the current user, role, and agency ID to controllers and services.

### Agency admin workflow
1. `AdminController` receives approve/reject/suspend requests.
2. `AgencyAdminService` updates agency status and the manager account state.
3. For approvals, manager fields like `approvedAt` and `approvedByAdminId` are set.

### Email verification workflow
1. `EmailVerificationService` creates a token and sends the email.
2. On verification, it sets `emailVerifiedAt` and activates the account for clients.

## Data model impact
- `AppUser` now includes `inscriptionDate` and `emailVerifiedAt`.
- `AgencyManager` now includes `licenceNumber`, `approvedAt`, `approvedByAdminId`, and `rejectionReason`.
- Repositories and controller IDs are now `String` to match the models ID type.
- Admin role value is now `SUPER_ADMIN` to match the models enum.

## Notes
- `CustomUserPrincipal` was removed and replaced by `CustomUserDetails`.
